## تحديد معدل الطلبات

تحديد معدل الطلبات هو أسلوب للتحكم في كمية الحركة التي يمكن لمستخدم أو عميل إرسالها إلى الخادم خلال فترة زمنية معينة.
يساعد هذا في منع إساءة الاستخدام والحماية من هجمات حجب الخدمة. ويمكن تنفيذ ذلك بطرق مختلفة، مثل token buckets أو leaky buckets أو fixed windows.

من خلال تحديد معدل الطلبات، يمكنك وضع حدود لعدد الطلبات التي يمكن للمستخدم إرسالها خلال إطار زمني محدد، مما يساعد على منع هجمات brute force وتقليل خطر تحميل الخادم فوق طاقته.

إذا لم تقم بإعداد تحديد معدل الطلبات، فقد يتمكن المهاجم من إرسال عدد كبير من الطلبات إلى خادمك، ما يرهقه وقد يتسبب في تعطله أو توقفه عن الاستجابة.
يمكن أن يؤدي ذلك إلى حجب الخدمة عن المستخدمين الشرعيين، ويمكن استخدامه أيضًا كتشتيت بينما يحاول المهاجم استغلال ثغرات أخرى في نظامك.

هذا مثال على محدد معدل بسيط مطبق في Next.js proxy. يتحقق من عدد الطلبات من عنوان IP معين ويمنع الطلبات التي تتجاوز الحدود المحددة.

```ts
import { NextRequest, NextResponse } from "next/server";
import { checkRateLimit } from "../../utils/rate-limit";

export default function RateLimiter(
  request: NextRequest,
): NextResponse | undefined {
  if (!/api\//.test(request.nextUrl.pathname)) {
    return undefined;
  }

  const forwardedFor = request.headers.get("x-forwarded-for");
  const ip = forwardedFor?.split(",")[0] || "unknown";
  const origin = request.headers.get("origin");

  const allowedOrigins =
    process.env.NODE_ENV === "development"
      ? [/^http:\/\/localhost:\d+$/, /^http:\/\/127\.0\.0\.1:\d+$/]
      : [/^https?:\/\/([a-z0-9-]+\.)?melvinjonesrepol\.com(:\d+)?$/];

  const isAllowed1 =
    !origin || allowedOrigins.some((pattern) => pattern.test(origin));

  if (!isAllowed1) {
    return NextResponse.json(
      { error: "Hehe you're going too far naah..." },
      { status: 403 },
    );
  }

  const maxRequest = /api\/(contact|report)/.test(request.nextUrl.pathname)
    ? 2
    : 10;

  const window = /api\/(contact|report)/.test(request.nextUrl.pathname)
    ? 60 * 60 * 1000
    : 5 * 60 * 1000;

  const isAllowed = checkRateLimit(ip, maxRequest, window);

  if (!isAllowed) {
    return NextResponse.json({ error: "Too many requests" }, { status: 429 });
  }

  return undefined;
}
```

دالة التحقق من معدل الطلبات:

```ts
const ipRequests: Record<string, { count: number; firstRequest: number }> = {};

export function checkRateLimit(
  ip: string,
  maxRequests: number,
  rateLimitWindow: number,
): boolean {
  const now = Date.now();

  if (!ipRequests[ip]) {
    ipRequests[ip] = { count: 1, firstRequest: now };
    return true;
  }

  const data = ipRequests[ip];
  if (now - data.firstRequest < rateLimitWindow) {
    if (data.count >= maxRequests) {
      return false; // ❌ over the limit
    }
    data.count++;
    return true;
  }

// إعادة تعيين النافذة
  ipRequests[ip] = { count: 1, firstRequest: now };
  return true;
}
```

## التحقق من المدخلات

استخدمنا أنواعًا مختلفة من التحقق من المدخلات لضمان أن البيانات المرسلة إلى تطبيقاتنا بالصيغة الصحيحة ولا تحتوي على أي كود خبيث.
يمكن أن يساعد هذا في منع ثغرات شائعة مثل SQL injection وcross-site scripting (XSS) وcommand injection. ويمكن تنفيذ التحقق من المدخلات باستخدام تقنيات متعددة مثل regular expressions وwhitelisting وblacklisting. من المهم التحقق من كل مدخلات المستخدم، بما في ذلك البيانات القادمة من النماذج ومعلمات الاستعلام والرؤوس.

قد يؤدي ضعف التحقق من المدخلات إلى ثغرات أمنية مثل SQL injection، حيث يمكن للمهاجم حقن SQL خبيث داخل استعلام، أو cross-site scripting (XSS)،
حيث يمكن للمهاجم حقن كود خبيث داخل صفحة ويب يتم تنفيذه بعد ذلك في متصفح المستخدم.

## المصادقة والتفويض

المصادقة هي عملية التحقق من هوية المستخدم أو العميل، بينما التفويض هو عملية تحديد الأفعال التي يُسمح للمستخدم أو العميل بتنفيذها.
إن تطبيق آليات قوية للمصادقة والتفويض يمكن أن يساعد في الحماية من الوصول غير المصرح به ويضمن أن المستخدمين لا يصلون إلا إلى الموارد المسموح لهم بها.

استخدام كلمات مرور قوية والمصادقة متعددة العوامل والتحكم بالوصول المعتمد على الأدوار يمكن أن يحسن أمان تطبيقاتك وأنظمتك.

## التحقق من البريد الإلكتروني ورقم الهاتف

يمكن أن يساعد التحقق من البريد الإلكتروني ورقم الهاتف في التأكد من أن معلومات الاتصال التي يقدمها المستخدمون صحيحة ويمكن استخدامها للتواصل.
يساعد هذا في منع الرسائل المزعجة والاحتيال والأنشطة الخبيثة الأخرى. ويمكن تنفيذ التحقق عبر إرسال رمز أو رابط تحقق إلى بريد المستخدم الإلكتروني أو رقم هاتفه.

يمكن أن يساعد حظر نطاقات البريد الإلكتروني المؤقتة والأرقام المعروفة أيضًا في منع الإساءة وتحسين جودة قاعدة المستخدمين.

## المصائد

هذه ما يُسمى خدعًا أو طعومًا صُممت لجذب المهاجمين وإيقاعهم. يمكن استخدامها لاكتشاف الهجمات وتحليلها، وكذلك لإبعاد المهاجمين عن الأهداف الأهم.
يمكن تنفيذ honeypots بطرق مختلفة، مثل مدخلات وهمية أو صفحات تسجيل دخول وهمية أو قواعد بيانات وهمية أو خدمات شبكية وهمية.

يمكن أن تساعد المصائد في تحسين أمان تطبيقاتك وأنظمتك من خلال توفير إنذار مبكر للهجمات والسماح لك بتحليل أنماط الهجوم وتقنياته.
وحظرها قبل أن تسبب ضررًا يمكن أن يساعد في منع الأذى عن أنظمتك وحماية مستخدميك.

## ReCAPTCHA أو hCAPTCHA

ReCAPTCHA وHCAPTCHA أدوات شائعة تستخدم لحماية المواقع من الرسائل المزعجة والإساءة عبر التحقق من أن المستخدم إنسان. ويمكنها منع الروبوتات الآلية من إرسال النماذج،
أو إنشاء الحسابات، أو تنفيذ إجراءات أخرى على موقعك.

يمكن أن يساعد تطبيق ReCAPTCHA أو HCAPTCHA في تحسين أمان تطبيقاتك وأنظمتك عبر تقليل خطر الرسائل المزعجة والإساءة. كما يمكن أن يحسن تجربة المستخدم بمنع التفاعلات غير المرغوبة مع موقعك.

هذا مثال على كيفية تطبيق ReCAPTCHA Enterprise وHoneypot في تطبيق Laravel. يتحقق الكود من حقلي البريد الإلكتروني وكلمة المرور، ويفحص حقل المصيدة، ويتحقق من استجابة ReCAPTCHA.

```php
$request->validate(
           [
               'email' => 'required|string',
               'password' => 'required|string',
               'username' => [new Honeypot()],
               'g-recaptcha-response' => ['required', new RecaptchaEnterprise('login')],
           ],
           [
               'g-recaptcha-response.required' =>
                   'Please try again later or contact support if the problem persists.',
           ],
       );
```

## بصمة المتصفح وتحليل الرؤوس

بصمة المتصفح وتحليل الرؤوس هما تقنيتان تُستخدمان لتحديد المستخدمين وتتبعهم بناءً على خصائص المتصفح والجهاز.
يمكن أن يساعد هذا في تحسين الأمان عبر تمكينك من اكتشاف النشاط المشبوه وحظره، مثل الروبوتات أو المستخدمين الخبثاء. كثير من الروبوتات والأتمتة تتبع الأنماط نفسها،
لذا من خلال تحليل الرؤوس والبصمات يمكنك التعرف عليها وحظرها قبل أن تسبب ضررًا.

على سبيل المثال، عادةً ما يكون للأتمتة نفس user agent، أو قد تفتقد بعض الرؤوس التي يمتلكها المتصفح العادي. ومن خلال تحليل هذه الأنماط يمكنك تحسين أمان تطبيقاتك وأنظمتك.

هذا مثال على فحص بسيط لمتصفح headless مطبق في Next.js proxy. يتحقق من رأس user agent بحثًا عن أنماط معروفة مرتبطة بمتصفحات headless ويحظر الطلبات المطابقة لها.

```ts
import { NextResponse } from "next/server";

const HEADLESS_PATTERNS = [
  "HeadlessChrome",
  "PhantomJS",
  "SlimerJS",
  "Puppeteer",
  "Playwright",
  "Chrome-Lighthouse",
] as const;

// حظر الروبوتات، والسماح للمتصفحات الحقيقية
export default function HeadlessBrowserCheck(
  req: Request,
): NextResponse | undefined {
  const ua = req.headers.get("user-agent") ?? "";
  const isHeadless = HEADLESS_PATTERNS.some((p) =>
    ua.toLowerCase().includes(p.toLowerCase()),
  );

  if (isHeadless) {
    return new NextResponse(
      JSON.stringify({ error: "Automated browsers are not allowed" }),
      {
        status: 403,
        headers: { "Content-Type": "application/json" },
      },
    );
  }

  return undefined;
}
```

## Dependabot وSnyk

Dependabot وSnyk أداتان تساعدانك على إبقاء الاعتماديات محدثة وآمنة. ويمكنهما فحص اعتمادياتك تلقائيًا بحثًا عن الثغرات وإرسال تنبيهات عند
اكتشاف ثغرات جديدة.

يمكن أن يساعد استخدام أدوات مثل Dependabot وSnyk في تحسين أمان تطبيقاتك وأنظمتك من خلال ضمان استخدام أحدث وأأمن إصدارات الاعتماديات.

## تشفير قاعدة البيانات (فقط إذا كانت لديك بيانات حساسة)

تشفير قاعدة البيانات هو عملية تشفير البيانات المخزنة في قاعدة البيانات لحمايتها من الوصول غير المصرح به. ويمكن أن يساعد هذا في تحسين أمان تطبيقاتك وأنظمتك عبر
ضمان حماية البيانات الحساسة حتى لو تم اختراق قاعدة البيانات.

يمكن تنفيذ تشفير قاعدة البيانات بطرق مختلفة، مثل تشفير البيانات أثناء التخزين، أو تشفير البيانات أثناء النقل، أو استخدام التشفير على مستوى الحقل.
من المهم التفكير بعناية في تأثير ذلك على الأداء واختيار الطريقة المناسبة لحالة الاستخدام الخاصة بك.

## OTP وMagic Links والمصادقة بدون كلمة مرور

OTP (One-Time Passwords) وMagic Links والمصادقة بدون كلمة مرور هي أساليب لا تتطلب من المستخدمين تذكر كلمة مرور وإدخالها.
يمكن لهذه الأساليب أن تحسن الأمان من خلال تقليل مخاطر الثغرات المرتبطة بكلمات المرور، مثل ضعف كلمات المرور وإعادة استخدامها وهجمات التصيد.

## سياسة أمان المحتوى (CSP)

سياسة أمان المحتوى (CSP) هي ميزة أمان تساعد في منع cross-site scripting (XSS) وهجمات حقن الكود الأخرى عبر تحديد مصادر المحتوى المسموح لصفحة الويب بتحميلها.
يمكن أن يساعد تطبيق CSP في تحسين أمان تطبيقاتك وأنظمتك عبر تقليل خطر XSS وهجمات حقن الكود الأخرى.
من المهم ضبط CSP بعناية للسماح فقط بالمصادر الموثوقة للمحتوى مع حظر المصادر المحتملة الخبث.

رغم أن ليس كل موقع يحتاج CSP، إلا أنه من الجيد تطبيقه إذا كان موقعك يحتوي على محتوى ينشئه المستخدمون أو إذا كنت تريد إضافة طبقة أمنية إضافية إلى تطبيقك.

## CORS

كابوس المطورين، Cross-Origin Resource Sharing (CORS)، هي ميزة أمنية تنفذها متصفحات الويب لمنع المواقع الخبيثة من إرسال طلبات غير مصرح بها إلى
نطاق مختلف. CORS موضوع معقد، لكنه مهم للمطورين لفهم آليته وكيفية ضبطه بشكل صحيح لضمان أمان تطبيقاتهم وأنظمتهم.

Improperly configured CORS can lead to security vulnerabilities, such as allowing unauthorized access to your API or exposing sensitive data to malicious websites.
It is important to carefully configure your CORS settings to allow only trusted origins and to restrict the types of requests that are allowed.

## Cloudflare أو أي CDN وWAF آخر

Not all Security Measures are implemented in the code, using a CDN (Content Delivery Network) and WAF (Web Application Firewall) can help improve the security of your applications and systems by providing additional layers of protection against various types of attacks,
such as DDoS attacks, SQL injection, and cross-site scripting (XSS).

CDNs can help improve the performance and security of your applications by caching content and distributing it across multiple servers, while WAFs can help protect your applications by filtering and monitoring incoming traffic for malicious activity.

## حظر عناوين IP والحظر الجغرافي

IP blocking and geo-blocking are techniques used to restrict access to your applications and systems based on the IP address or geographic location of the user.
This can help improve security by blocking access from known malicious IP addresses or from regions that are not relevant to your business.

## تعلّم الهجمات القائمة على الهندسة الاجتماعية والتصيد

Majority of attacks are not technical, but rather social engineering and phishing attacks. Learning about these types of attacks can help you understand how attackers manipulate people to gain access to sensitive information or systems.
They pretend to be someone else, such as a trusted friend, colleague, or organization, to trick victims into revealing sensitive information or performing actions that can compromise security.

They will do this by sending emails, creating fake websites, events or social media profiles, or even making phone calls to trick victims into revealing sensitive information or performing actions that can compromise security.
Don't install software from untrusted sources, and be cautious when clicking on links or downloading attachments from unknown senders.
