## مشكلة المتصفحات الحالية

كثير من متصفحات الويب لأجهزة Android تكون مثقلة بميزات قد تبطئ الأداء وتستهلك موارد أكثر. بالإضافة إلى ذلك، قد لا تعطي بعض المتصفحات خصوصية المستخدم الأولوية،
ما يؤدي إلى مخاوف بشأن تتبع البيانات والأمان. ونتيجة لذلك، يجد المستخدمون أنفسهم عالقين مع متصفحات لا تلبي احتياجاتهم من حيث السرعة والأمان وتجربة التصفح السلسة.

## مشروعي الأول

أحد مشاريعي في بدايات تعلم البرمجة كان بناء متصفح ويب لأجهزة Android. لماذا Webvium Browser؟ لماذا ليس آلة حاسبة أو شيئًا آخر؟ في ذلك الوقت كانت لدي مشكلة مع المتصفحات الحالية، فهي بطيئة وغير محسنة للأجهزة الضعيفة. أردت إنشاء متصفح خفيف وسريع وآمن، ويقدم تجربة تصفح ممتازة على أي جهاز Android.
وكان التحدي الآخر هو بناء متصفح يمكنه العمل على الأجهزة الضعيفة من دون استهلاك موارد كثيرة. أردت إنشاء متصفح يمكن لأي شخص استخدامه بغض النظر عن قدرات جهازه.
هذا يعني صفر اعتماديات، وJava خالصة وAndroid SDK. أردت إنشاء متصفح يمكنه منافسة اللاعبين الكبار في السوق، بهدف واحد: اجعله أخف ما يمكن وأسرع ما يمكن.

هذا مثال على قاعدة كود Webvium Browser، وهو متصفح ويب بسيط يحمّل موقعًا.

```xml
<!-- res/layout/activity_main.xml -->
<?xml version="1.0" encoding="utf-8"?>
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <WebView
        android:id="@+id/webView"
        android:layout_width="match_parent"
        android:layout_height="match_parent" />

</RelativeLayout>
```

```java
// src/main/java/com/mrepol742/Webvium/MainActivity.java
package com.mrepol742.Webvium;

import android.os.Bundle;
import android.webkit.WebView;
import android.webkit.WebViewClient;
import androidx.appcompat.app.AppCompatActivity;

public class MainActivity extends AppCompatActivity {

    WebView webView;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        webView = findViewById(R.id.webView);

        // تفعيل JavaScript (اختياري لكنه غالبًا ضروري)
        webView.getSettings().setJavaScriptEnabled(true);

        // فتح الروابط داخل WebView بدلًا من المتصفح الخارجي
        webView.setWebViewClient(new WebViewClient());

        // تحميل موقع ويب
        webView.loadUrl("https://www.melvinjonesrepol.com");
    }

    // التعامل مع زر الرجوع للعودة في سجل WebView
    @Override
    public void onBackPressed() {
        if (webView.canGoBack()) {
            webView.goBack();
        } else {
            super.onBackPressed();
        }
    }
}
```

```xml
<!-- AndroidManifest.xml -->
<uses-permission android:name="android.permission.INTERNET" />
```

بناء تطبيق بحجم أقل من 1MB تحدٍّ حقيقي، بل كابوس، بل شيء مستحيل، لكنني فعلته على أي حال. درست بعمق كيف يعمل Android، والسمات، وكيفية تحسين الكود ليكون أكثر كفاءة ممكنة.
تم إطلاق النسخة الأولى من Webvium Browser في وقت ما من أكتوبر 2018، وكان متصفح ويب بسيطًا لا أكثر، لكنه كان قفزة مهمة لي كمطور مبتدئ. البحث في Google، والتنقل إلى أعمق زوايا Stack Overflow،
وقراءة التوثيق الرسمي لـ Android SDK كانت روتيني اليومي في ذلك الوقت، بلا مساعد ذكاء اصطناعي ولا GitHub Copilot، فقط عمل جاد وإصرار على التعلم وبناء شيء رائع.

تم إصدار النسخة الأولى بميزات أساسية مثل تحميل صفحات الويب والتنقل ذهابًا وإيابًا والإشارات المرجعية. وكان الحجم كله أقل من 5MB، وهذا ليس شيئًا يُفتخر به، لذا بدأت استراتيجية التعلم العميق هنا. عندها
ركزت بعمق على تحسين قاعدة الكود وتقليل حجم التطبيق. قضيت ساعات في تحليل الكود والبحث عن طرق لجعله أكثر كفاءة، وهو ما أدى في النهاية إلى تقليل الحجم بشكل كبير.
أزلت الاعتماديات، وحسّنت تخطيط XML، وأعدت هيكلة الكود ليصبح أكثر كفاءة. وبعد عدة دورات من التحسين، تمكنت من خفض حجم التطبيق إلى أقل من 500KB، وهو إنجاز مهم جدًا لمتصفح ويب.
