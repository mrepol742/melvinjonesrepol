## ما هو هجوم استكشاف المعرفات؟

هجوم استكشاف المعرفات هو نوع من الهجمات يحاول فيه المهاجم تخمين معرفات الموارد في النظام أو تعدادها.
يمكن أن يتم ذلك بزيادة قيم المعرفات أو إنقاصها، أو باستخدام أنماط شائعة لتخمينها. الهدف من هذا الهجوم هو الوصول غير المصرح به إلى الموارد أو
جمع معلومات عن النظام. كما أن استخدام معرفات متسلسلة للموارد يجعل من السهل معرفة العدد الإجمالي للموارد في النظام مثل
عدد المستخدمين والطلبات وسلال التسوق وغيرها، كما يسهل على المهاجمين تخمين المعرفات الصحيحة والوصول إلى موارد لا ينبغي لهم الوصول إليها.

هذا مسار هجوم شائع في تطبيقات الويب، خصوصًا تلك التي تستخدم معرفات متسلسلة للموارد.
على سبيل المثال، إذا كان تطبيق ويب يستخدم معرفات متسلسلة لحسابات المستخدمين، فقد يحاول المهاجم الوصول إلى حسابات أخرى عبر زيادة أو إنقاص قيمة المعرف في الرابط.

```php
Schema::table('users', function (Blueprint $table) {
    $table->id();
    $table->string('name');
    $table->string('email')->unique();
    $table->timestamps();
});
```

```php
// مثال على هجوم استكشاف المعرفات
// لنفترض أن لدينا رابطًا مثل هذا: https://example.com/user/123
// قد يحاول المهاجم الوصول إلى https://example.com/user/124 وhttps://example.com/user/125 وما إلى ذلك
```

## كيف نمنع هجوم استكشاف المعرفات؟

لمنع هجوم استكشاف المعرفات، يمكنك استخدام الاستراتيجيات التالية:

1. **استخدم معرفات غير متسلسلة**: بدلًا من استخدام معرفات متسلسلة، استخدم UUIDs أو ULIDs أو slug أو اسم المستخدم أو أي أسلوب آخر غير متسلسل.
2. **نفّذ تفويضًا صحيحًا**: تأكد من أن المستخدمين لا يصلون إلا إلى الموارد التي يملكون صلاحية عرضها.
3. **تحديد المعدل**: نفّذ rate limiting لمنع الهجمات الآلية.
4. **التحقق من المدخلات**: تحقّق من كل المدخلات للتأكد من أنها ضمن النطاقات والصيغ المتوقعة.
5. **معالجة الأخطاء**: قدّم رسائل خطأ عامة بدلًا من كشف معلومات عن البنية الداخلية للنظام.

بتطبيق هذه الاستراتيجيات، يمكنك تقليل خطر هجمات استكشاف المعرفات بشكل كبير وحماية تطبيقك من الوصول غير المصرح به.

## في Laravel

يمكنك استخدام route model binding للتعامل مع التفويض تلقائيًا ومنع استكشاف المعرفات. على سبيل المثال:

```php
// In your routes/web.php
Route::get('/user/{user}', function (App\Models\User $user) {
    // The $user variable will automatically be resolved to the User model instance
    // You can also add authorization logic here to ensure the user has access to this resource
    return view('user.profile', ['user' => $user]);
});
```

باستخدام route model binding، يمكنك التأكد من معالجة المعرفات الصحيحة فقط، وأن المستخدمين لا يصلون إلا إلى الموارد المسموح لهم بعرضها، وبذلك تمنع هجمات استكشاف المعرفات.

## استكشاف معرفات السلة والطلب

في تطبيقات التجارة الإلكترونية، من الشائع استخدام معرفات متسلسلة للسلال والطلبات. لمنع استكشاف المعرفات في هذا السياق، يمكنك استخدام UUIDs أو معرفات أخرى غير متسلسلة للسلال والطلبات. بهذه الطريقة، حتى لو حاول المهاجم تخمين المعرفات، فلن يتمكن من الوصول إلى سلال أو طلبات المستخدمين الآخرين.

```php
Schema::table('orders', function (Blueprint $table) {
    $table->uuid('order_id')->primary();
    $table->unsignedBigInteger('user_id');
    $table->decimal('total_amount', 8, 2);
    $table->timestamps();
});
```

يمكنك أيضًا استخدام المعرف وUUID معًا لأسباب تتعلق بالأداء، بحيث يُستخدم المعرف للإشارات الداخلية ويُستخدم UUID للمعرفات الظاهرة للعامة.

```php
Schema::table('orders', function (Blueprint $table) {
    $table->id();
    $table->uuid('order_uuid')->unique();
    $table->unsignedBigInteger('user_id');
    $table->decimal('total_amount', 8, 2);
    $table->timestamps();
});
```

خيار آخر بدل UUID هو استخدام حزمة ULID، التي تولّد معرفات قابلة للترتيب يمكن استخدامها كمفاتيح أساسية في قاعدة البيانات. هذا قد يوفر أداءً أفضل من UUID التقليدي مع الاستمرار في منع استكشاف المعرفات.

```php
Schema::table('orders', function (Blueprint $table) {
    $table->ulid('order_id')->primary();
    $table->unsignedBigInteger('user_id');
    $table->decimal('total_amount', 8, 2);
    $table->timestamps();
});
```

لا تنسَ إخفاء المعرف المتسلسل عن الـ API الظاهر للعامة، وإظهار UUID فقط للمستخدمين. بهذه الطريقة، حتى لو حاول المهاجم تخمين المعرف المتسلسل، فلن يتمكن من الوصول إلى أي موارد من دون UUID المقابل.

```php
// في النموذج الأساسي
protected $hidden = ['id'];
```

باستخدام UUIDs لمعرفات الطلبات، يمكنك منع المهاجمين من تخمين معرفات الطلبات الصحيحة والوصول بسهولة إلى طلبات المستخدمين الآخرين. بالإضافة إلى ذلك،
يجب أن تطبق فحوصات تفويض صحيحة للتأكد من أن المستخدمين لا يصلون إلا إلى طلباتهم الخاصة.

---

هجوم استكشاف المعرفات هو مسار هجوم شائع قد يؤدي إلى الوصول غير المصرح به إلى الموارد داخل تطبيق ويب.
وباستخدام معرفات غير متسلسلة، وتطبيق تفويض صحيح، واتباع أفضل الممارسات في التحقق من المدخلات ومعالجة الأخطاء،
يمكنك تقليل خطر هذا الهجوم بشكل كبير وحماية تطبيقك من الوصول غير المصرح به.
