# Floating Console Extension وDisable Ctrl Shift C Extension

أحيانًا أفضل أدوات المطورين ليست الأطر الضخمة أو المنصات المعقدة.

أحيانًا يكون الأمر مجرد إصلاح شيء مزعج يقطع سير عملك خمسين مرة في اليوم.

هكذا تقريبًا وُلدت هاتان الإضافتان للمتصفح.

إحداهما تبقي الـ console مرئيًا أثناء التصفح.  
والأخرى تنقذ مستخدمي الطرفية من فتح DevTools بالخطأ كل بضع دقائق.

مشكلات صغيرة. وراحة كبيرة بشكل مفاجئ.

## Floating Console Extension

كانت الفكرة وراء Floating Console Extension بسيطة.

لماذا ينبغي على المطورين الاستمرار في فتح وإغلاق DevTools فقط لفحص السجلات؟

خصوصًا على الشاشات الصغيرة، يبدو فتح DevTools وكأنك تضحي بنصف نافذة المتصفح لآلهة التصحيح.

لذلك بنيت إضافة خفيفة تمنح المطورين console عائمًا مباشرة على الصفحة.

Console يبقى متاحًا أثناء التصفح من دون إعادة تحجيم النوافذ أو تبديل الألسنة باستمرار.

لأن التصحيح لا ينبغي أن يبدو كإعادة ترتيب الأثاث كل خمس دقائق.

## لماذا بنيته

أثناء التطوير، كنت أجد نفسي باستمرار:

- أفتح DevTools
- أفحص السجلات
- أغلق DevTools
- أعيد فتح DevTools بعد ثانيتين
- أكرر الدورة حتى أشك في اختياراتي الحياتية

تزيل Floating Console Extension هذا الاحتكاك عبر إبقاء السجلات مرئية بطريقة أكثر راحة بكثير.

وهي مفيدة بشكل خاص لـ:

- تصحيح الواجهة الأمامية
- مراقبة السجلات أثناء اختبار واجهة المستخدم
- الإعدادات ذات الشاشة الصغيرة
- سير العمل متعدد الشاشات
- المطورين الذين لديهم بالفعل 37 تبويبًا مفتوحًا ويرفضون إغلاق أي منها

وهو تقريبًا نحن جميعًا.

## الميزات

تشمل الوظائف الحالية:

- floating overlay داخل المتصفح
- رؤية سهلة للـ console
- تصميم خفيف
- وصول سريع للتصحيح
- أقل قدر من الإعاقة لسير العمل

كان الهدف إبقاء الأمر بسيطًا ومفيدًا بدلًا من تحويله إلى "developer suite" مترهل آخر.

أحيانًا القليل هو الكثير فعلًا.

## Disable Ctrl Shift C Extension

إذا كنت تستخدم terminals Linux بانتظام، فأنت تعرف هذا الألم بالفعل.

تضغط:

```bash
Ctrl + Shift + C
```

متوقعًا نسخ نص من terminal داخل المتصفح...

ثم يفتح DevTools فجأة كضيف غير مدعو.

مرة أخرى.

ومرة أخرى.

ومرة أخرى.

هذا التعارض في الاختصار أزعج المطورين لسنوات عبر المتصفحات. :contentReference[oaicite:2]{index=2}

لذا بدلًا من الاستسلام، بنيت إضافة متصفح خصيصًا لتعطيل اختصار DevTools `Ctrl + Shift + C`.

مشكلة بسيطة. حل بسيط.

بصراحة، هذه الإضافة موجودة لأن الذاكرة العضلية أقوى من الإعدادات الافتراضية للمتصفح.

## لماذا هذا مهم

للمطورين الذين يستخدمون:

- browser terminals
- Cloud IDEs
- جلسات SSH
- workflows Linux
- Docker web terminals
- خوادم بعيدة

يُستخدم `Ctrl + Shift + C` عادةً لعمليات النسخ.

لكن المتصفحات تحب تخصيص الاختصار نفسه لفحص DevTools.

وهذا يخلق دورة لا تنتهي من فتح DevTools بالخطأ أثناء العمل العادي. وقد اشتكى مستخدمو Mozilla من هذا السلوك لسنوات.

تصلح هذه الإضافة هذا الإزعاج مباشرة.

لا إعداد معقد.  
لا إعادة بناء المتصفحات من المصدر.  
لا حيل غريبة مخبأة داخل ملفات الإعداد.

فقط ثبّتها وتوقف عن محاربة اختصارات لوحة المفاتيح.

## بناء أدوات صغيرة تحل مشاكل حقيقية

بُنيت هاتان الإضافتان من العقلية نفسها.

تجربة المطور مهمة.

حتى المقاطعات الصغيرة تصبح مرهقة عندما تتكرر مئات المرات كل أسبوع.

أفضل الأدوات ليست دائمًا الأكثر بهرجة.  
بل تلك التي تزيل الاحتكاك بهدوء وتسمح للمطورين بالتركيز على العمل الحقيقي.

أو على الأقل تقلل عدد المرات التي نفتَح فيها DevTools عن طريق الخطأ ونحن نحاول نسخ نص.

هذا وحده يبدو تقدمًا بالفعل.
